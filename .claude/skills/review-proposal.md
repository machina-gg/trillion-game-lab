# 企画書レビュー

## 概要

Planner が作成した企画書 PR をレビューし、review-notes.md に結果を記録する。

## 手順

### Step 1: 企画書の読み込み

proposals/inbox/{name}/proposal.md を読み、以下を把握する:
- エレベーターピッチ（コンセプトの要約）
- 課題定義と解決策
- ターゲットユーザー
- 差別化ポイント
- MVP 機能一覧
- 市場規模
- ビジネスモデル
- データソース・調査方法

### Step 2: ファクトチェック（WebSearch で検証）

企画書内のデータを WebSearch で直接検証する:
- 市場規模の数値が妥当か（複数ソースでクロスチェック）
- 競合情報が正確か（公式サイト・レビューサイトで確認）
- トレンド情報が最新か（直近 6 ヶ月以内のデータか）
- ソース URL が有効か（リンク切れがないか）
- ソースの信頼度は十分か（公式発表・業界レポート・大手メディア等）

**重要**: 企画書に記載されたソース URL を WebFetch で確認し、データの正確性を検証する。

### Step 3: レビューノート作成（review-notes.md）

proposals/inbox/{name}/review-notes.md を作成する:

```markdown
# レビューノート: {プロダクト名}

## レビュー日
{YYYY-MM-DD}

## レビュアー
Editor

## 総合評価
APPROVE / REQUEST_CHANGES

## レビュー観点

### 1. 論理性
- 評価: OK / 要改善
- コメント: {課題→解決策の論理が通っているか}

### 2. 完全性
- 評価: OK / 要改善
- コメント: {必要なセクションが全て埋まっているか}

### 3. データ裏付け
- 評価: OK / 要改善
- コメント: {主張にリサーチデータの裏付けがあるか}

### 4. 実現可能性
- 評価: OK / 要改善
- コメント: {技術的に実装可能か、リソースは現実的か}

### 5. 差別化
- 評価: OK / 要改善
- コメント: {競合との違いが明確で説得力があるか}

### 6. 市場規模
- 評価: OK / 要改善
- コメント: {TAM/SAM/SOM が妥当か}

## 良い点
- {具体的に記載}

## 改善点
1. {指摘}: {理由と改善案}
2. {指摘}: {理由と改善案}

## ファクトチェック結果
| 項目 | 企画書の値 | 検証結果 | 一致 |
|------|-----------|---------|------|
| {項目} | {値} | {WebSearch で確認した情報} | OK / NG |
```

### Step 4: PR にコメント

```bash
# レビュー結果を PR に投稿
gh pr review {PR番号} --approve --body "企画書レビュー完了。詳細は review-notes.md を参照。"

# または
gh pr review {PR番号} --request-changes --body "改善点があります。review-notes.md を確認してください。"
```

## レビュー観点の詳細

| 観点 | チェック内容 |
|------|------------|
| 論理性 | 課題→解決策の論理が通っているか。飛躍がないか |
| 完全性 | 必要なセクション（課題、ターゲット、MVP、市場規模、ビジネスモデル）が全て埋まっているか |
| データ裏付け | 主張にリサーチデータの裏付けがあるか。数値の出典が明記されているか |
| 実現可能性 | 技術的に実装可能か。MVP の範囲が現実的か。月額 $250 の制約内か |
| 差別化 | 競合との違いが明確で説得力があるか。「なぜ既存プロダクトではダメか」が説明できているか |
| 市場規模 | TAM/SAM/SOM が妥当か。CAGR のソースは信頼できるか |

## REQUEST_CHANGES 時の優先度付け

改善点が複数ある場合、優先度を付けて Planner に伝える:

| 優先度 | 基準 | 例 |
|--------|------|-----|
| **P0（必須）** | これがないと企画として成立しない | 差別化が不明確、市場データがない |
| **P1（推奨）** | 品質向上に直結する | リスク分析が不足、ペルソナが曖昧 |
| **P2（任意）** | あると良いが必須ではない | 表現の改善、追加データ |
