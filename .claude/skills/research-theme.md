# テーマリサーチ

## 概要

PM からテーマを受け取り、WebSearch/WebFetch でデータ収集し、research/{theme}/ にドキュメントを作成する。

## 手順

### Step 1: テーマの理解

- Issue の要件を確認する
- リサーチの目的・スコープを把握する
- 不明点があれば PM にエスカレーションする

### Step 2: ディレクトリ作成

```bash
mkdir -p research/{theme-slug}/
```

テーマスラッグは英語の kebab-case で命名する（例: `ai-code-editor`, `saas-dashboard`）

### Step 3: 市場分析（market-analysis.md）

WebSearch で市場規模・成長率・主要プレイヤーを調査する。

**調査項目:**
- TAM（Total Addressable Market）: 市場全体の規模
- SAM（Serviceable Addressable Market）: 自社がアプローチ可能な市場
- SOM（Serviceable Obtainable Market）: 現実的に獲得可能な市場
- CAGR（年平均成長率）
- 主要プレイヤー（3社以上）

**テンプレート:**
```markdown
# 市場分析: {テーマ}

## 市場規模

| 指標 | 値 | ソース |
|------|-----|--------|
| TAM | ${N}B | [ソース名](URL) |
| SAM | ${N}B | [ソース名](URL) |
| SOM | ${N}M | 推定 |
| CAGR | {N}% | [ソース名](URL) |

## 主要プレイヤー

| 企業 | シェア | 特徴 |
|------|--------|------|
| {名前} | {N}% | {概要} |

## 市場トレンド

- {トレンド 1}
- {トレンド 2}

## データ取得日
{YYYY-MM-DD}
```

### Step 4: トレンドレポート（trend-report.md）

WebSearch でトレンド・バイラル情報を収集する。

**調査項目:**
- テクノロジートレンド（新技術、AI活用等）
- 消費者トレンド（行動変化、ニーズの変化等）
- 規制・政策トレンド（該当する場合）
- 直近6ヶ月のニュース・動向

**テンプレート:**
```markdown
# トレンドレポート: {テーマ}

## テクノロジートレンド

### {トレンド名}
- 概要: {説明}
- 影響: {プロダクトへの影響}
- ソース: [記事名](URL)

## 消費者トレンド

### {トレンド名}
- 概要: {説明}
- データ: {裏付けデータ}
- ソース: [記事名](URL)

## 注目ニュース（直近6ヶ月）

| 日付 | ニュース | ソース |
|------|---------|--------|
| {YYYY-MM} | {概要} | [リンク](URL) |

## データ取得日
{YYYY-MM-DD}
```

### Step 5: 競合分析（competitor-analysis.md）

WebSearch/WebFetch で競合の機能・価格・ユーザー数を調査する。

**調査項目（競合ごと）:**
- プロダクト名・URL
- 主要機能
- 料金プラン
- ユーザー数/収益（公開されている場合）
- 強み・弱み

**テンプレート:**
```markdown
# 競合分析: {テーマ}

## 競合一覧

### {競合 1}
- URL: {URL}
- 概要: {1行説明}
- 主要機能: {機能リスト}
- 料金: {プラン概要}
- ユーザー数: {数値またはN/A}
- 強み: {リスト}
- 弱み: {リスト}

### {競合 2}
...

## 機能比較マトリックス

| 機能 | {競合1} | {競合2} | {競合3} | Gap |
|------|---------|---------|---------|-----|
| {機能1} | O | O | X | - |
| {機能2} | X | O | X | ブルーオーシャン |

## Gap 分析（ブルーオーシャン機会）

- {機会 1}: {説明}
- {機会 2}: {説明}

## データ取得日
{YYYY-MM-DD}
```

### Step 6: ソース一覧（sources.md）

全ソースURLを一覧化し、信頼度を評価する。

**信頼度評価:**
- ★★★: 公式発表、政府統計、学術論文
- ★★: 業界レポート、大手メディア、専門メディア
- ★: ブログ、個人分析、SNS

**テンプレート:**
```markdown
# ソース一覧: {テーマ}

## 一次ソース（公式発表・統計）

| # | ソース名 | URL | 信頼度 | 取得日 | 使用箇所 |
|---|---------|-----|--------|--------|---------|
| 1 | {名前} | {URL} | ★★★ | {日付} | market-analysis.md |

## 二次ソース（レポート・メディア）

| # | ソース名 | URL | 信頼度 | 取得日 | 使用箇所 |
|---|---------|-----|--------|--------|---------|
| 1 | {名前} | {URL} | ★★ | {日付} | trend-report.md |
```

### Step 7: PR 作成

全ドキュメント作成後、PR を作成する。

```bash
gh pr create --repo machina-gg/trillion-game-lab \
  --title "feat: {テーマ}のリサーチ" \
  --body "## Summary
- market-analysis.md: 市場規模・成長率の調査
- trend-report.md: テクノロジー・消費者トレンドの収集
- competitor-analysis.md: 競合{N}社の分析
- sources.md: ソース{N}件の一覧化

Closes #{issue-number}"
```

## WebSearch/WebFetch の使い方

- 英語と日本語の両方で検索する
- 「{テーマ} market size 2025」「{テーマ} competitors」等のクエリを使う
- 公式サイトの情報を WebFetch で取得する
- ソースの日付に注意（古いデータは明記する）
- 複数ソースでクロスチェックする（重要データは2つ以上のソースで裏付け）

## 品質基準

- [ ] 全データにソースURLがある
- [ ] 市場規模は TAM/SAM/SOM で推定
- [ ] 競合は最低3社分析
- [ ] トレンドは直近6ヶ月のデータを含む
- [ ] sources.md に全URLが一覧化されている
- [ ] 信頼度評価が付いている
