# タスク分解チェックリスト

タスクを開始する前に PM が実施する。

## 1. 影響ファイルの洗い出し

- [ ] このタスクで変更が必要なファイルを全て列挙した
- [ ] 各ファイルの所有ロールを特定した

| 変更ファイル | 所有ロール | 担当者 |
|---|---|---|
| (記入) | Researcher / Planner / Editor / PM | (記入) |

## 2. ロール割当

- [ ] 自分の担当外のファイルがある場合、該当ロールのエージェントに委譲する
- [ ] **PM は research/ proposals/ を直接編集しない**（エージェントを起動して委譲）
- [ ] **PM 管轄ファイル（CLAUDE.md, .claude/, .github/）の変更が必要な場合は PM が実施**
- [ ] 複数ロールが関わる場合、タスクをロールごとに分割した

## 3. 委譲判断

| 状況 | アクション |
|------|----------|
| research/ のリサーチが必要 | Researcher エージェントを起動 |
| proposals/ の企画書が必要 | Planner エージェントを起動 |
| レビューが必要 | Editor エージェントを起動 |
| CLAUDE.md / .claude/ / .github/ の変更が必要 | PM が自分で実施 |
| 複数ロール | 各ロールのエージェントを起動し、タスクを分割して委譲 |

## 4. タスクサイズ判定

| サイズ | 基準 | フロー |
|--------|------|--------|
| **S** | 1ファイル / 明確な修正 | Issue → Branch → 作業 → PR → マージ |
| **M** | 2-5ファイル / 既存パターン踏襲 | Issue → Branch → 作業 → PR → レビュー → マージ |
| **L** | 6+ファイル / リサーチ→企画の一連の流れ | Issue → Researcher → Planner → Editor レビュー → PR → マージ |

## 5. 作業開始

- [ ] 全ての委譲先が決まった
- [ ] TaskCreate でタスクを作成し、owner を割り当てた
- [ ] 自分が担当するタスクのみ作業を開始する
